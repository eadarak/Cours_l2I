Comment installer un serveur DHCP dans Ubuntu et Debian

Aaron Kili5 mars 2018March 25, 2017 Catégories DHCP 14 commentaires

Protocole de configuration d'hôte dynamique (DHCP) est un protocole réseau utilisé pour permettre aux ordinateurs hôtes de se voir attribuer automatiquement des adresses IP et des configurations réseau associées à partir d'un serveur.

L'adresse IP attribuée par un serveur DHCP au client DHCP se trouve sur une “ bail ”, le délai de location varie normalement en fonction de la durée pendant laquelle un ordinateur client est susceptible d'exiger la connexion ou la configuration DHCP.

Comment fonctionne DHCP?

Voici une description rapide du fonctionnement réel du DHCP:

  • Une fois qu'un client ( est configuré pour utiliser DHCP ) et connecté à un réseau démarre, il envoie un DHCPDISCOVER paquet sur le serveur DHCP.
  • Lorsque le serveur DHCP reçoit le DHCPDISCOVER demande paquet, il répond avec un DHCPOFFER paquet.
  • Ensuite, le client obtient le DHCPOFFER paquet, et il envoie un DHCPREQUEST paquet au serveur indiquant qu'il est prêt à recevoir les informations de configuration réseau fournies dans le DHCPOFFER paquet.
  • Enfin, après réception du serveur DHCP DHCPREQUEST paquet du client, il envoie le DHCPACK paquet montrant que le client est désormais autorisé à utiliser l'adresse IP qui lui est attribuée.

Dans cet article, nous vous montrerons comment configurer un serveur DHCP dans Ubuntu / Debian Linux, et nous exécuterons toutes les commandes avec le commande sudo pour obtenir des privilèges utilisateur racine.

Configuration de l'environnement de test

Nous allons utiliser l'environnement de test suivant pour cette configuration.

Serveur DHCP - Ubuntu 16.04 
Clients DHCP - CentOS 7 et Fedora 25

Étape 1: Installation du serveur DHCP dans Ubuntu

1. Exécutez la commande ci-dessous pour installer le package du serveur DCHP, qui était auparavant connu sous le nom de serveur dhcp3.

$ sudo apt install isc-dhcp-server

2. Une fois l'installation terminée, modifiez le fichier /etc / default / isc-dhcp-server pour définir les interfaces que DHCPD devrait utiliser pour servir les demandes DHCP, avec l'option INTERFACES.

Par exemple, si vous voulez que le démon DHCPD soit écouté eth0, réglez-le comme ceci:

INTERFACES = "eth0"

Et aussi apprendre à configurer une adresse IP statique pour l'interface ci-dessus.

Étape 2: configuration du serveur DHCP dans Ubuntu

3. Le fichier de configuration DHCP principal est /etc/dhcp/dhcpd.conf, vous devez ajouter toutes vos informations réseau à envoyer aux clients ici.

Et, il existe deux types d'instructions définies dans le fichier de configuration DHCP, ce sont:

  • paramètres – spécifiez comment effectuer une tâche, si vous devez effectuer une tâche ou quelles options de configuration réseau envoyer au client DHCP.
  • déclarations – définir la topologie du réseau, indiquer les clients, proposer des adresses aux clients ou appliquer un groupe de paramètres à un groupe de déclarations.

4. Maintenant, ouvrez et modifiez le fichier de configuration principal, définissez les options de votre serveur DHCP:

$ sudo vi /etc/dhcp/dhcpd.conf

Définissez les paramètres globaux suivants en haut du fichier, ils s'appliqueront à toutes les déclarations ci-dessous ( spécifiez les valeurs qui s'appliquent à votre scénario ):

nom de domaine d'option "tecmint.lan";
option serveurs de noms de domaine ns1.tecmint.lan, ns2.tecmint.lan;
temps de location par défaut 3600; 
temps de location max 7200;
faisant autorité;

5. Maintenant, définissez un sous-réseau; ici, nous allons configurer DHCP pour 192.168.10.0/24 Le réseau LAN ( utilise des paramètres qui s'appliquent à votre scénario ).

sous-réseau 192.168.10.0 masque de réseau 255.255.255.0 {
        routeurs d'options 192.168.10.1;
        sous-réseau-mâche 255.255.255.0;
        option recherche de domaine            "tecmint.lan";
        option serveurs de noms de domaine 192.168.10.1;
        plage 192.168.10.10 192.168.10.100;
        192.168.10.110 192.168.10.200;
}

Étape 3: configurer l'IP statique sur la machine client DHCP

6. Pour attribuer une adresse IP fixe ( statique ) à un ordinateur client particulier, ajoutez la section ci-dessous où vous devez spécifier explicitement ses adresses MAC et l'IP à attribuer statiquement:

nœud centos hôte {
	 Ethernet matériel 00: f0: m4: 6y: 89: 0g;
	 adresse fixe 192.168.10.105;
 }

hôte fedora-node {
	 Ethernet matériel 00: 4g: 8h: 13: 8h: 3a;
	 adresse fixe 192.168.10.106;
 }

Enregistrez le fichier et fermez-le.

7. Ensuite, démarrez le service DHCP pour le moment et activez-le pour démarrer automatiquement à partir du prochain démarrage du système, comme ceci:

----------- SystemD ----------- 
$ sudo systemctl start isc-dhcp-server.service
$ sudo systemctl active isc-dhcp-server.service


------------ SysVinit ----------- 
$ sudo service isc-dhcp-server.service start
$ sudo service isc-dhcp-server.service activ

8. Ensuite, n'oubliez pas d'autoriser le service DHCP ( Le démon DHCPD écoute sur le port 67 / UDP ) sur le pare-feu comme ci-dessous:

$ sudo ufw allow 67 / udp
$ sudo ufw reload
$ sudo ufw show

Étape 4: Configuration des machines client DHCP

9. À ce stade, vous pouvez configurer les ordinateurs de vos clients sur le réseau pour recevoir automatiquement les adresses IP du serveur DHCP.

Connectez-vous aux ordinateurs clients et modifiez le fichier de configuration de l'interface Ethernet comme suit ( prenez note du nom / numéro de l'interface ):

$ sudo vi / etc / network / interfaces

Et définissez les options ci-dessous:

auto eth0
iface eth0 inet dhcp

Enregistrez le fichier et quittez. Et redémarrez les services réseau comme ( ou redémarrer le système ):

----------- SystemD ----------- 
$ sudo systemctl redémarrer la mise en réseau

------------ SysVinit ----------- 
redémarrage du réseau de services sudo $

Vous pouvez également utiliser l'interface graphique sur une machine de bureau pour effectuer les paramètres, définir la méthode sur automatique ( DHCP ) comme indiqué dans la capture d'écran ci-dessous ( Bureau Fedora 25 ).

À ce stade, si tous les paramètres sont correctement configurés, votre machine cliente doit recevoir automatiquement des adresses IP à partir du serveur DHCP.

C'est ça! Dans ce didacticiel, nous vous avons montré comment configurer un serveur DHCP dans Ubuntu / Debian. Partagez vos réflexions avec nous via la section des commentaires ci-dessous. Si vous utilisez la distribution basée sur Fedora, suivez comment configurer un serveur DHCP dans CentOS / RHEL.

      Vous pouvez le faire, mais votre configuration est fausse. Vous devez en savoir plus sur le masquage de sous-réseau. Votre masque de réseau devrait permettre à tous ces clients de s'intégrer dans la plage. Après cela, vous pouvez ajouter tous les sous-réseaux que vous souhaitez et dont vous avez besoin.

      Un autre problème que vous rencontrerez est de diffuser des tempêtes avec autant de clients sur la même partition réseau. Ce sera encore pire si vous utilisez des communications sans fil.

      Si vous voulez continuer, quelque chose comme ça devrait faire l'affaire:

      Un sous-réseau de 255.255.252.0 donnera = 192.168.16.1 – 192.168.19.254

      Non, vous ne pouvez pas simplement commencer à partir de .10.x car ce sous-réseau est au milieu d'un autre.

      sous-réseau 192.168.16.0 masque de réseau 255.255.252.0 {
        routeurs d'options 192.168.16.1;
        option subnet-mask 255.255.252.0;
        option recherche de domaine "tecmint.lan";
        option serveurs de noms de domaine 192.168.16.1;
        plage 192.168.16.10 192.168.16.255;
        192.168.17.10 192.168.17.255;
        plage 192.168.18.10 192.168.18.255;
        plage 192.168.19.10 192.168.19.254;
}

      Cela semble bizarre pour la plupart des gens, mais le 192.168.16.255 est une adresse utilisable dans ce sous-réseau. Cela vous donnera les 10 premières adresses IP de chaque plage de nombres à être statiques si vous le souhaitez, mais maximise sur l'adresse utilisable dans le serveur dhcp.

      Je recommanderais également d'envisager de diviser cela en 2 peut-être 4 serveurs. DHCP travaille sur une course. Le premier serveur à répondre peut émettre l'adresse IP. Par conséquent, si l'un de vos serveurs se casse, les autres peuvent toujours servir vos clients.
      Les gens n'aiment pas quand “ Internet ” se casse.
